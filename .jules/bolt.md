## 2026-07-02 - Optimizing List Filtering in React
**Learning:** Found instances where lists were being filtered using `useEffect` to synchronize a separate `filteredItems` state variable whenever the original list or search query changed. This is a common React anti-pattern that triggers an unnecessary double-render (one for the input change, and a second when the effect fires to update the filtered list). Additionally, string operations like `.toLowerCase()` were placed inside the `.filter()` loop, causing them to execute `N` times instead of just once.
**Action:** When working on lists that depend on another piece of state (like a search query), always use `useMemo` to compute the derived state directly instead of using `useState` and `useEffect`. Always hoist redundant computations (like `searchQuery.toLowerCase()`) outside the loop to optimize performance.
