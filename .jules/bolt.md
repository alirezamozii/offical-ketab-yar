## 2024-05-27 - Remove Unnecessary Server-to-Client Data Serialization
**Learning:** In Next.js App Router, explicitly serializing data using `JSON.parse(JSON.stringify(data))` before passing it from Server Components to Client Components is a performance anti-pattern. Next.js handles the serialization natively and securely for plain JavaScript objects, so manually re-serializing just adds unnecessary CPU overhead on both the server (to stringify/parse) and causes code bloat. This is only necessary if passing functions or un-serializable objects like Dates (which Next.js warns about), but for simple data like arrays of plain objects, it's a net loss.
**Action:** When passing plain object data (like book lists or user profiles) from Server to Client components, pass the objects directly. Ensure the data contains no methods or non-serializable types, and rely on Next.js native handling instead of manual serialization.
