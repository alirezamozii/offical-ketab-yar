## 2025-02-18 - Replacing useEffect state sync with useMemo for list filtering
**Learning:** When synchronizing state derived from props or other state (like a `filteredWords` list derived from `words` and `searchQuery`), using `useEffect` combined with `useState` creates an anti-pattern. This results in an unnecessary double-render cycle because the component renders once with the old state, the `useEffect` fires to update the derived state, and then the component renders again. This is particularly harmful for performance when the list is large.
**Action:** Use `useMemo` to derive the state directly during the render cycle. This completely eliminates the double-render and significantly improves performance. Ensure that any transformations, such as converting search queries to lowercase, are hoisted out of the filter loop inside the `useMemo` callback to minimize redundant computations on each item.
