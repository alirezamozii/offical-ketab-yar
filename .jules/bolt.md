## 2026-07-04 - Prevent Unnecessary Render Cycles on Derived Array State
**Learning:** The codebase contains a common React performance anti-pattern where derived state arrays (like filtered lists from API responses) are synchronized via `useEffect` into a separate `useState`. This causes an unnecessary double-render cycle for every filter or input keystroke. When fixing these, it's also highly efficient to hoist constant calculations (like `toLowerCase()` on search inputs) outside of the `.filter()` loop rather than recalculating them per iteration.
**Action:** Always replace `useState` + `useEffect` synchronizations with a dynamically derived `useMemo` computation for lists and arrays to prevent triple or double-render cascades, and strictly pull out any static loop variables.
