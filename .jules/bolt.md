## 2026-05-08 - Unnecessary JSON Serialization in Next.js Server Components
**Learning:** In Next.js App Router, when passing data from Server Components to Client Components, plain JavaScript objects containing primitive values do not need to be serialized using the `JSON.parse(JSON.stringify(data))` hack. If the data is already mapped to clean objects without Dates, functions, or complex class instances, Next.js handles the serialization automatically. Using the hack simply wastes CPU cycles on unnecessary stringification and parsing.
**Action:** Always verify if data being passed to Client Components is already a plain object. If it is, pass it directly. Remove any existing `JSON.parse(JSON.stringify())` calls unless dealing with specific ORM objects (like Mongoose documents) or class instances that Next.js cannot serialize natively.
