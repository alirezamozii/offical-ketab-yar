## 2024-07-02 - Accessible Loading State for Follow Toggle
**Learning:** Icon-only toggle buttons (like "Follow") present a dual challenge: they need to communicate their current state visually, provide context to screen readers on what *will* happen when clicked, and indicate loading states when interacting with the backend. Using a loading spinner over an icon without an aria-label leaves screen reader users stranded, especially during network latency. Adding dynamic aria-labels ("Follow" vs "Unfollow" depending on state) ensures action intent is clear.
**Action:** When creating icon-only async toggles, wrap the icon swap in a ternary checking `loading`, and apply dynamic `aria-label`s based on the expected outcome of the click (e.g., `isFollowing ? 'Unfollow' : 'Follow'`).
