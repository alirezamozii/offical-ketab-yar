## 2024-05-24 - Stateful Async Icon Button Feedback
**Learning:** When dealing with buttons that execute stateful asynchronous actions (like "Follow/Unfollow"), screen readers need dynamic labels that reflect the action the user *will* take (e.g., `aria-label={isFollowing ? "Unfollow" : "Follow"}`). Furthermore, immediately switching the ARIA label or completely dropping the visible icon structure during a loading phase can cause layout shifts or confusion. Using a spinner like `Loader2` instead of the action icon perfectly signals background work while the `disabled` state prevents multi-click regressions.
**Action:** When improving micro-UX on interactive lists or cards, always ensure action buttons are equipped with both conditional `aria-label`s based on the current entity state and visual loading indicators mapped directly to the component's async handlers.
